## 1. EXECUTIVE SUMMARY

### EN
"Explain to My Kid" / "Wytłumacz Dziecku" targets a **verified, underserved market gap** at the intersection of:
- Children's education
- Visual/infographic content
- AI-generated, age-adaptive explanations
- Polish-language digital content

**3 independent research perspectives agree**: the Polish-language market is a **blue ocean** — no direct competitor exists. Global competitors solve pieces of the problem; none combine infographic-first + AI generation + age filtering + Polish language + content safety in a single product.

**Aggregate rating across perspectives:**
| Perspective | Rating | Rationale |
|-------------|--------|-----------|
| Parent | 8.4/10 | Solves daily pain point; AI accuracy is main risk |
| Child/Student | 8.2/10 | Format is perfect; lacks animation and interactivity |
| Teacher/Educator | 8.2/10 | High potential; needs curriculum alignment + review layer |
| **COMBINED** | **8.3/10** | Strong go-to-market opportunity |

### PL
"Wytłumacz Dziecku" trafia w **zweryfikowaną, niedostatecznie obsługiwaną lukę rynkową** na przecięciu edukacji dzieci, treści wizualnych/infograficznych, wyjaśnień generowanych przez AI i polskojęzycznych treści cyfrowych. ## 4. USER RESEARCH BY PERSPECTIVE

### 4.1 Parent Pain Points (Top 10)
1. "My child asks a question I can't explain simply enough" — daily, emotionally significant
2. "Everything online is either too simple or too scary/inappropriate"
3. "Can't trust Google results to be age-appropriate"
4. "My child learns visually but educational sites are text walls"
5. "Don't know if content is appropriate for MY child's specific age"
6. "Child gets distracted / rabbit-holes into inappropriate content"
7. "Best content is in English only" (barrier for Polish families)
8. "Want to start a conversation but don't know how to begin"
9. "My neurodiverse child isn't served by standard content"
10. "Can't verify if information is accurate / not misinformation"

### 4.2 Child/Student Needs (Top 8)
1. Portrait-mobile-first format (95% of mobile use is portrait)
2. Visual-first content (processes images 60,000× faster than text)
3. Age-appropriate vocabulary (hates being talked down to OR talked over)
4. Polish language (currently gets Wikipedia — impossible to understand)
5. Safe content (stumbling on inappropriate content causes real distress)
6. Age-calibrated: "that's baby stuff" frustration is real
7. Shareable format (social currency for ages 10-16)
8. Interactive elements (quizzes, animations multiply engagement 2-4x)

### 4.3 Teacher/Educator Pain Points (Top 10)
1. Inappropriate ads on "free" educational platforms
2. Login/account creation wall before class can use it
3. English-only quality content in Polish schools
4. Not aligned to Polish curriculum (Podstawa Programowa)
5. Wrong reading level / cognitive level for age group
6. Desktop-only, not mobile/smartboard-friendly
7. No quick-summary/anchor format (infographic wins over 15-min video)
8. AI-generated content not reviewed/trusted
9. No classroom integration (shareable links, QR codes, lesson plans)
10. Too expensive for school budgets (50-200 PLN per-pupil IT budget limit)

## 5. CORE FEATURE IDEAS + RATINGS

> Rating scale: 0.0–10.0 | Criteria: User impact × Market differentiation × Feasibility

### F1: Portrait-Only Infographic Format ⭐ 9.5/10
**Description:** All content displayed as portrait-mode infographics (9:16 ratio, max 450px wide), optimized for mobile consumption. No landscape mode fallback.
**User value:** 10/10 — matches how 95% of children use their phones; mirrors Instagram/TikTok format they're trained on
**Differentiation:** 10/10 — NO existing educational platform does portrait-first
**Feasibility:** 9/10 — CSS `aspect-ratio: 9/16` + design system
**Rating: 9.5/10** — Non-negotiable core format decision. DO NOT compromise.

---

### F2: Age-Group Content Filtering (Under 13 / 13+) ⭐ 9.2/10
**Description:** Same topic explained at two depth levels. Under-13: simple, concrete, shorter. 13+: more nuanced, abstract, longer. Age group stored in localStorage (no login required).
**User value:** 9/10 — parents + children both cite age-mismatch as a top frustration
**Differentiation:** 9/10 — Newsela does it for text; nobody does it for infographics
**Feasibility:** 9/10 — Generate 2 versions per topic via Gemini; tag in DB
**Rating: 9.2/10** — Critical differentiator. Must be in v1.

---

### F3: REGEX + Semantic Content Safety Layer ⭐ 9.0/10
**Description:** Multi-layer content filtering: REGEX blocklist (phase 1) + Gemini safety classification (phase 2) + human review flag (phase 3). Applies to: search queries, user-generated questions, AI-generated content before publishing.
**User value:** 9/10 — both parents and teachers cite content safety as top-3 concern
**Differentiation:** 8/10 — REGEX alone is standard; semantic + human layer is differentiated
**Feasibility:** 9/10 — Well-established patterns; Gemini has built-in safety filters
**Rating: 9.0/10** — Legal and trust requirement. Must ship in v1.

---

### F4: Browse Without Login ⭐ 9.0/10
**Description:** Full content library browsable without account creation. Age preference stored in localStorage. Only AI generation and bookmarks require (optional) account.
**User value:** 9/10 — #1 classroom adoption barrier is account creation; parents also hate sign-up walls
**Differentiation:** 9/10 — Most premium platforms require login
**Feasibility:** 10/10 — Default architecture decision; simplifies GDPR compliance
**Rating: 9.0/10** — Critical for teacher adoption. Also safest GDPR approach.

---

### F5: 1000+ Pre-Generated Infographic Library ⭐ 8.8/10
**Description:** Curated library of 1000+ most-frequently-asked questions by children, each with a kid-friendly infographic explanation. Built via Gemini API + design templates + human review.
**User value:** 9/10 — parents and kids want immediate answers, not just generation capability
**Differentiation:** 8/10 — Content is the moat; quality + quantity matter
**Feasibility:** 8/10 — Gemini generation + design system makes this achievable; human review is the bottleneck
**Rating: 8.8/10** — Core content library is the product. Prioritize this above all features.

---

### F6: AI-Powered "Ask Any Question" Generation ⭐ 8.5/10
**Description:** Users (parents or kids) type any question. Gemini generates a new infographic explanation calibrated to the selected age group. Generated content saved to DB (with review flag) for future reuse.
**User value:** 9/10 — parents' most-requested feature in EdTech surveys
**Differentiation:** 9/10 — No kid-safe platform does real-time AI infographic generation
**Feasibility:** 8/10 — Gemini API + infographic template rendering pipeline
**Risk:** AI inaccuracy; must flag AI-generated content as "AI draft - under review"
**Rating: 8.5/10** — High-impact feature; needs accuracy safeguards.

---

### F7: Topic Search + Filter System ⭐ 8.5/10
**Description:** Search by topic name (full-text), category (Science, History, Technology, etc.), age group, popularity (most viewed), most liked, most recently added.
**User value:** 8/10 — standard discovery mechanism; parents and teachers need curriculum browsing
**Differentiation:** 8/10 — Age+category combined filter is not standard
**Feasibility:** 9/10 — PostgreSQL full-text search with Polish language support (pg_trgm + Polish stemming)
**Rating: 8.5/10** — Core utility feature. Polish-language search is a technical requirement.

---

### F8: "Most Liked" / Social Signals ⭐ 7.8/10
**Description:** Like/save button (no login required via localStorage; syncs if logged in). Public "Most Popular" section. "Trending this week" section.
**User value:** 8/10 — parents want to know what other families are exploring; social proof drives trust
**Differentiation:** 7/10 — Standard feature but valuable in this context
**Feasibility:** 9/10 — Simple upvote counter + Redis for trending calculation
**Rating: 7.8/10** — Include in v1; low effort, high signal value.

## 6. ENHANCEMENT OPPORTUNITIES + RATINGS

> These go beyond the core concept. Rated for impact × effort × differentiation.

### E1: Animated SVG/Lottie Infographics ⭐ 9.0/10
**Description:** Replace static infographics with micro-animated versions. Small CSS/Lottie animations (e.g., a cell dividing, Earth rotating, water cycle flowing). Not full video — subtle motion only.
**User value:** 9.5/10 (children) — static vs. animated engagement difference is 2-4x in research
**Effort:** Medium-High — requires design system overhaul; Lottie JSON animations per infographic
**When:** v2
**Rating: 9.0/10** — Most impactful enhancement for child engagement. Build design system for it from day 1.

---

### E2: Post-Infographic "Test Yourself" Quiz ⭐ 8.8/10
**Description:** 3-question multiple-choice quiz auto-generated by Gemini from the infographic content. Immediate feedback. Optional (not gated). Score visible but not stored without account.
**User value:** 9/10 — retrieval practice increases retention by 50% (Roediger & Karpicke)
**Differentiation:** 8/10 — Wonderopolis has quizzes; doing it well for infographics is novel
**Effort:** Medium — Gemini can generate quizzes from infographic text
**When:** v1.5 or v2
**Rating: 8.8/10** — Strong engagement + educational validity. Easy to build on top of existing Gemini pipeline.

---

### E3: "Explain This News Article for My Child" Tool ⭐ 8.7/10
**Description:** Parent pastes any news URL → Gemini extracts content → generates age-appropriate kid-safe summary infographic. Output is NOT saved publicly; parent-only private use.

| DSA (EU Digital Services Act) | Platform with UGC or recommendations | Age verification for restricted content; no minor targeting in ads | Must implement |
| EU AI Act (2024) | AI-generated educational content | High-risk category; human oversight required for educational AI | Human review layer required |
| WCAG 2.1 Level AA | All public-facing content | Alt text, contrast, keyboard nav, screen reader compatibility | Build in from day 1 |

### 8.2 Safest Architecture Decisions
1. **Browse-without-login** — eliminates GDPR consent requirement for most users
2. **No behavioral advertising** — ever
3. **Server-side content safety filtering** — before ANY content is served
4. **Age-group preference in localStorage only** — not a profile, not PII
5. **Human review layer** before any AI content goes public
6. **Appoint a Child Protection Coordinator** (Koordynator ds. ochrony dzieci) before launch

## 12. UI/UX DESIGN RESEARCH

> Full design spec: [UI_UX_DESIGN.md](./UI_UX_DESIGN.md)

### 12.1 Visual Style Verdict

The "light-comic" intuition is **validated with one important refinement**: use **"Warm Editorial Illustration"** rather than pure cartoon. Research breakdown:

| Style | Ages 6–12 | Ages 13–16 | Overall |
|-------|-----------|------------|---------|
| Full cartoon/comic | 9/10 | 5/10 | 7/10 |
| **Warm editorial illustration (graduated)** | **9/10** | **8/10** | **9/10** |
| Flat/minimal design | 5/10 | 7/10 | 6/10 |
| Photographic | 4/10 | 6/10 | 5/10 |
| Aggressive gamification visual | 7/10 (6-12) | 2/10 (13-16) | 4/10 |

**Warm editorial illustration** = illustrated characters with personality + clean typographic hierarchy + infographic elements drawn in the same art style. Expressiveness and saturation are *tuned down* as age increases — one design system, calibrated spectrum.

Key validation: DK Find Out and National Geographic Kids both use this exact approach and are the most-cited reference designs in children's educational content research.

### 12.2 Mascot: "Ryś" (Lynx) ⭐ 9/10

**Recommendation: A stylized Polish lynx named "Ryś"**

| Criterion | Score | Notes |
|-----------|-------|-------|
| Cultural differentiation | 10/10 | Białowieża lynx = recognized Polish natural symbol |
| Competitor uniqueness | 10/10 | Not used by any major EdTech competitor globally |
| Visual design potential | 9/10 | Ear tufts + spots + bright eyes = expressive at all sizes |
| Name appeal | 9/10 | "Ryś" = animal name + Polish diminutive in one |
| Age-range scalability | 9/10 | Full cartoon (6-8) → icon mark (13+) |
| Research backing | 9/10 | Animal mascots outperform human mascots for cross-cultural child engagement |

Animal mascots outperform human characters because: no ethnic/gender ambiguity, can be exaggerated without "uncanny valley", flexible across cultural contexts (Joan Ganz Cooney Center research).

### 12.3 Color Palette ⭐ 9/10

**"Warm Science" Palette**

| Role | Color | Hex | Rationale |
|------|-------|-----|-----------|
| Brand anchor | Cerulean | `#1A9BBF` | Knowledge, exploration — cross-cultural |
| Discovery accent | Amber | `#F59E0B` | Energy, warmth, "aha moments" |
| Emotion accent | Coral | `#E8634A` | Warmth, emotional topics |
| Background | Warm cream | `#FFFBF5` | Quality-printed-book feel; NOT pure white (causes visual stress) |
| Body text | Warm black | `#1C1917` | Never pure #000 — harsh on screens |

**Why this palette beats alternatives**: Teal/cerulean is universally associated with knowledge and exploration. Amber differentiates from Duolingo (green) and Khan Academy (blue/purple). Cream background is Polish parents' trust register — "quality book".

**Category colors (9 topics):** each topic gets a distinct color (`#0D9488` Science, `#B45309` History, `#2563EB` Tech, `#16A34A` Nature, `#E11D48` Body, `#7C3AED` Society, `#1E40AF` Space, `#EA580C` Emotions, `#4338CA` Philosophy)

### 12.4 Typography ⭐ 9/10

| Font | Role | Why |
|------|------|-----|
| **Baloo 2** | Display/headings (Under-13) | Rounded, warm; validated Polish diacritic support |
| **Lexend** | Body text (all ages) | Specifically designed for reading ease; dyslexia-validated; Polish support |
| **Inter** | Display/headings (13+) | Editorial, clean; best-in-class Polish diacritic rendering |
| **Lora** | Optional serif accent (13+) | Editorial sophistication for teen register |

Critical rule: `latin-ext` subset required for full Polish character coverage (ą ć ę ł ń ó ś ź ż).

### 12.5 Navigation Pattern ⭐ 9/10

**Bottom tab bar, 4 tabs, always visible.**

Research finding: hamburger menus are invisible to children under 9 and reduce task completion by 40% in under-13 users (NNG). Bottom navigation is the correct pattern for this platform.

Tabs: Główna (Home) · Tematy (Browse) · Szukaj (Search) · Ulubione (Saved)

Touch targets: 60px for Under-13, 48px for 13+. Text labels always paired with icons.

### 12.6 Infographic Layout: "5-Part Portrait Scroll" ⭐ 9.5/10

Every infographic follows one structure (scroll-triggered reveals):
1. **Hero Zone** — title + Ryś + hook sentence
2. **Why It Matters** — 1-2 sentences, amber accent
3. **How It Works** — main explanation, illustrated diagram, 3-5 key points
4. **Real-World Example** — concrete analogy, "It's like..." box
5. **Key Facts + Quiz CTA** — 2-3 surprising facts + "Test yourself"

Limits: max 7 sections, max 4-6 visual elements per section, 20/80 text/visual ratio (Under-8) → 50/50 (13+).

### 12.7 Animation System ⭐ 8.5/10

- **Under-13**: Spring physics (slight overshoot) — Framer Motion spring / CSS custom easing
- **13+**: Cubic-bezier ease-out, no overshoot
- **Mascot**: Lottie JSON (6 emotional states, 24fps)
- **Infographic reveals**: Intersection Observer + CSS transitions (scroll-triggered section entry)
- **Always honor `prefers-reduced-motion`** — no exceptions

### 12.8 Accessibility Requirements ⭐ 9.5/10

- WCAG 2.1 AA: legal minimum; build from day 1
- Dyslexia mode toggle (Lexend + increased spacing, localStorage)
- ADHD-friendly: no auto-play, no countdown timers, chunked content, paginated browse
- SVG infographics: `<title>` + `<desc>` + full text DOM fallback for screen readers
- Focus styles: visible `outline: 3px solid #2563EB` (Tailwind resets these — must override)
- Color blindness: never red/green only; all category labels have text + color

### 12.9 Key Anti-Patterns to Avoid (Research-Backed)

| Anti-Pattern | Why It Fails |
|--------------|-------------|
| Hamburger menu as primary nav | Invisible to children under 9; 40% lower task completion |
| Pure white `#FFFFFF` background | Visual stress in ~15% of children |
| Aggressive gamification visuals | Polish parents classify as "game" and restrict access |
| Auto-playing animations during reading | Reduces comprehension (Mayer Coherence Effect) |
| Icon-only navigation | Children need text labels to understand icons |
| Justified text | Creates "rivers" that impair dyslexic readers |
| All-caps body text | Destroys word-shape recognition used by developing readers |
| Parallax scrolling | Vestibular disorder trigger; confuses scroll position |
| Mascot auto-triggering mid-reading | The Clippy problem — drives immediate user annoyance |
| Infinite scroll | ADHD trap; makes it impossible to find previously seen content |
